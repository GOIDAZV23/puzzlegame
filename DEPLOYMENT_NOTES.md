# DEPLOYMENT_NOTES

## Публикация проекта

Проект представляет собой статический сайт, поэтому достаточно разместить файлы на любом хостинге, поддерживающем HTML/CSS/JS.

### GitHub Pages
1. Убедитесь, что репозиторий содержит `index.html` в корне.
2. На странице репозитория выберите `Settings` → `Pages`.
3. В разделе `Source` установите ветку `main` и папку `/ (root)`.
4. Нажмите `Save` и дождитесь обновления.

### Важные моменты
- Проверяйте относительные пути: `style.css` и `script.js` должны находиться рядом с `index.html`.
- Если сайт не обновился сразу, подождите несколько минут, пока GitHub Pages пересоберёт сайт.
- Используйте только `HTTPS` для внешних ресурсов, чтобы избежать блокировок браузером.

## Локальный запуск
1. Откройте `index.html` напрямую в браузере.
2. Проблемы с отображением обычно связаны с путями к файлам.

## Альтернативные варианты развертывания
- Загрузить файлы на любой статический хостинг (Netlify, Vercel, Surge и др.).
- Раздать через локальный веб-сервер (`python -m http.server` или аналогичный).
